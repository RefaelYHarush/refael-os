# עשה לפי סדר חשיבות – REFAEL OS

מסמך אחד עם כל הפעולות לפי עדיפות. סמן ☑ אחרי ביצוע.

---

## עדיפות 1 – וידוא שהכל עובד (עכשיו)

| # | פעולה | איך | סטטוס |
|---|--------|-----|--------|
| 1.1 | **התקנת תלויות** | `cd refael-os` → `npm install` | ☑ |
| 1.2 | **הרצת האתר** | `npm run dev` → פתח http://localhost:5173 (או הפורט שמופיע) | ☐ (פתח בדפדפן) |
| 1.3 | **הרצת בדיקות** | `npm run test:run` – לוודא שהכל עובר | ☑ (6 טסטים עוברים) |
| 1.4 | **בנייה לפרודקשן** | `npm run build` – לוודא שאין שגיאות | ☑ |

**אם 1.2 או 1.4 נכשלים** – תקן לפני שממשיך. אם 1.3 נכשל – אפשר להמשיך (הטסטים אופציונליים) אבל עדיף לתקן.

## עדיפות 2 – אבטחה והגדרות (פעם אחת)

| # | פעולה | איפה / איך | סטטוס |
|---|--------|------------|--------|
| 2.1 | **מדיניות סיסמה** | Supabase Dashboard → Authentication → Providers → Email. וודא אורך מינימלי (למשל 8) ומורכבות אם רלוונטי. ראה [PASSWORD_POLICY.md](PASSWORD_POLICY.md). | ☐ |
| 2.2 | **Redirect URLs באיפוס סיסמה** | Supabase → Authentication → URL Configuration → Redirect URLs. וודא ש־`https://refael-os.vercel.app/reset-password` (ודומיין הפרודקשן שלך) ברשימה. | ☐ |
| 2.3 | **.env מקומי** | אם אתה מפתח מקומית: העתק `.env.example` ל־`.env` ומלא ערכים מ־Supabase. | ☑ (הקובץ קיים – מלא ערכים אם צריך) |
